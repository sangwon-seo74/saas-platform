# 슈퍼 관리자(Super Admin) 추가 기능 로드맵

> 작성일: 2026-05-22
> 작성 맥락: 슈퍼어드민 사이트의 기본 기능 구현이 완료된 후, 다음 단계로 무엇을 추가하면 좋을지에 대한 제안.

---

## 🔥 가장 임팩트 큰 것 (1~2주 안에 가치 회수)

| 기능 | 효과 |
|---|---|
| **테넌트 가장(Impersonation)** — "이 테넌트로 로그인" 버튼 | 고객 지원/디버깅이 10배 빨라짐. 가장 자주 요청되는 슈퍼어드민 기능 중 하나 |
| **인보이스 자동 발행 cron** | 매월 결제 주기에 맞춰 `tenant_invoices`를 자동 INSERT (현재는 수동 처리) |
| **만료 D-30/14/7 자동 이메일** | 갱신율 직접 향상. 슈퍼어드민이 매일 만료 임박 확인하는 수고 제거 |
| **슈퍼어드민 액션 감사 로그** | 누가/언제/무엇을 변경했는지 `audit_logs`에 자동 기록 (현재는 로그인만 기록) |

---

## 🛡️ 보안 강화 (실서비스 전 필수에 가까움)

- **IP 화이트리스트** — 슈퍼어드민은 사무실/VPN IP에서만 접근 (.env로 관리)
- **2FA(TOTP) 강제** — 슈퍼어드민 계정만이라도 TOTP 의무화
- **로그인 Rate limiting** — 같은 IP/이메일에서 5분 내 5회 실패 시 차단 (audit_logs로 카운트 가능)
- **위험 작업 재인증** — 강제 해지/임시 비밀번호 발급 시 비밀번호 재확인 프롬프트
- **임시 비밀번호 이메일 전송** — 화면 노출 외에 안전한 채널 제공 (Resend 연동 시 자동)

---

## 💰 수익 분석 강화 (의사결정 데이터)

- **이탈률(Churn Rate) 추세** — 월별 churn% 라인 차트, 현재 MRR 추이 옆에 나란히
- **Cohort Retention** — 1월 가입자의 6개월 후 유지율 같은 매트릭스
- **이탈 사유 통계** — `cancel_reason` 워드 클라우드 + 카운트
- **LTV/ARPU 카드** — 대시보드에 평균 고객 생애가치 한 줄 추가
- **무료→유료 전환율** — 체험 → active 전환 비율

---

## ⚙️ 자동화/배치

- **결제 실패 자동 재시도** — failed 인보이스를 N시간 후 재시도 (cron)
- **6개월 비활성 사용자 자동 비활성화** — `last_login_at` 기반
- **테넌트 일일 사용량 스냅샷** — 사용자 수, 고객사 수, 메시지 수 시계열 저장
- **주간 운영 요약 이메일** — 월요일 아침 슈퍼어드민에게 자동 발송

---

## 📊 UX 개선 (저비용·고만족)

- **정렬 가능한 테이블 헤더** — MRR/만료일/가입일 클릭 정렬 (모든 목록 페이지)
- **CSV Export 버튼** — 테넌트/인보이스 목록 다운로드
- **상단 알림 센터** — Bell 아이콘이 있지만 미동작 → 미납/만료/오류 알림 모음
- **글로벌 검색(Cmd+K)** — 모든 페이지에서 테넌트/사용자/이메일 즉시 검색
- **페이지네이션 UI** — 현재 100건 fixed, 그 이상이면 보이지 않음 (이미 일부 페이지에 truncation banner는 있음)

---

## 🤝 고객 지원 도구

- **테넌트 운영 메모** — 슈퍼어드민이 테넌트별 자유 메모 기록 (전화 통화 기록 등)
- **사용자 강제 로그아웃** — Supabase `auth.admin.signOut(userId)` 호출
- **사용자 역할 변경** — 특정 사용자의 role(admin↔manager↔sales) 변경
- **테넌트별 우선순위/등급 태깅** — VIP/Standard/Risk 등 색상 태그

---

## 💳 결제 운영

- **인보이스 PDF 생성** — 결제 완료 시 자동 생성하거나 다운로드 버튼
- **세금계산서 발행 상태 추적** — 한국 SaaS 운영에 자주 필요
- **할인/크레딧 부여** — 환불 대신 차회 결제에서 차감되는 크레딧
- **인보이스 메모를 일반 사용자도 보기 옵션** — 결제 실패 사유 안내

---

## ⭐ 추천 — "다음에 한다면 이 3개"

1. **테넌트 가장(Impersonation)** — 운영 효율 개선 압도적
2. **인보이스 자동 발행 cron + 만료 알림 이메일** — 수동 작업 제거 + 갱신율 향상
3. **슈퍼어드민 액션 감사 로그** — 감사·책임 추적, 향후 모든 기능의 기반

## 📍 현재까지 구현 완료된 슈퍼어드민 기능 (참고)

- **Dashboard**: MRR/ARR 카드, 활성 테넌트 수, 만료 임박, 미납 인보이스, 플랜 분포, 6개월 MRR 추이
- **Tenants**: 목록(검색/상태 필터), 신규 등록(초대 메일 발송), 상세 5탭(기본정보·구독·결제·사용량·사용자), 정보 편집, 활성/비활성 토글, 임시 비밀번호 발급, 초대 메일 재발송, 서비스 강제 해지
- **Subscriptions**: 목록(상태 필터), 만료 임박 알림, MRR 합계
- **Invoices**: 목록(상태 필터), 상세, 수동 결제 완료/환불 처리, 운영 메모
- **Plans**: 카드 뷰, 가격/제한 편집, 활성/비활성 토글, MRR 기여 차트
- **System Logs**: 접속 로그(audit_logs 실데이터, 검색/필터), 공지/점검(생성/편집/토글/삭제)
- **Auth 통합**: 로그인 성공/실패 자동 audit_logs 기록 + `users.last_login_at` 자동 갱신
